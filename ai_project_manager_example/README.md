# Example: Filled-in Project Files

This folder shows what the `ai_project_manager_kickstart/project/` files
look like when filled in for a real project.

The example project is a personal recipe book web app — simple enough to
follow, complex enough to show the files working together.

## What's here

```text
project/
  brief.md           What we're building — filled in by hand.
  architecture.md    Stack and structure — generated by AI, then reviewed.
  conventions.md     Style and naming rules — captured during first tasks.
  backlog.md         Task list — generated by AI, reordered by hand. Shows mixed status.
  file-map.md        Key files — populated as code was written.
  decision-log.md    Design decisions — appended during each task's design phase.
```

## How this relates to the kickstart pack

The templates in `ai_project_manager_kickstart/project/` are blank
versions of these files. The guided setup in
`ai_project_manager_kickstart/init.md` (or the web control panel at
`ai_project_manager_kickstart/index.html`) walks you through filling
them in.

These examples show what "filled in" looks like after a few tasks have
been completed — not just after init. Notice that `backlog.md` has
mixed status markers and `decision-log.md` has multiple entries.
