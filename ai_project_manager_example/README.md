# Example: Filled-in Project Files

This folder shows what all the kickstart files look like when filled in
for a real project.

The example project is a personal recipe book web app — simple enough to
follow, complex enough to show the files working together.

## What's here

```text
Permanent behavioral contracts (live in project root):
  AGENTS.md            AI agent rules — populated from the template.
  UI-STANDARDS.md      UI rules — only the token systems section shown.
  DEV-INFRASTRUCTURE.md  Build, dev server, scripts — populated from the template.
  project-README.md    Root README — named with prefix to avoid collision
                       with this folder's README. In a real project this
                       would be README.md at the project root.

Living project memory (in project/):
  project/brief.md           What we're building — filled in by hand.
  project/architecture.md    Stack and structure — generated by AI, then reviewed.
  project/conventions.md     Style and naming rules — captured during first tasks.
  project/backlog.md         Task list — generated by AI, reordered by hand.
  project/file-map.md        Key files — populated as code was written.
  project/decision-log.md    Design decisions — appended during each task.
```

## How this relates to the kickstart pack

The templates in `ai_project_manager_kickstart/` and
`ai_project_manager_kickstart/project/` are blank versions of these
files. The guided setup in `ai_project_manager_kickstart/init.md` (or
the web control panel at `ai_project_manager_kickstart/index.html`)
walks you through filling them in.

These examples show what "filled in" looks like after a few tasks have
been completed — not just after init. Notice that:

- `AGENTS.md` has all template placeholders replaced with concrete rules
- `DEV-INFRASTRUCTURE.md` documents the actual npm scripts and toolchain
- `backlog.md` has mixed status markers (done, in progress, todo)
- `decision-log.md` has multiple entries from different task sessions
- `conventions.md` includes Documentation and Tooling sections
