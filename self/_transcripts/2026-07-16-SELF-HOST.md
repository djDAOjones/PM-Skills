# The collision that forces the structure

In a normal consuming project the deployment occupies `pm_skills/project/` + root `AGENTS.md`/`UI-STANDARDS.md`/`DEV-INFRASTRUCTURE.md`. In this repo **those exact paths are the product** — `pm_skills/` must stay a pristine distributed tree ([MANIFEST.md](file:///Users/joe/Library/CloudStorage/OneDrive-OurWiltonTrust/_Joe%20OE%20Drive/4_Work/2026-04-12%20pm-skills%20Framework%20Repository/pm_skills/MANIFEST.md:0:0-0:0) constraint), and the root templates ship with `<!-- CUSTOMISE -->` placeholders intact ([README.md](file:///Users/joe/Library/CloudStorage/OneDrive-OurWiltonTrust/_Joe%20OE%20Drive/4_Work/2026-04-12%20pm-skills%20Framework%20Repository/README.md:0:0-0:0) states they are "not filled-in contracts for this repository"). So the repo's own deployment needs a parallel home that mirrors the standard shape without touching product paths.

# Options

- **A — `self/` mirrored deployment (recommended).** One top-level directory holds the entire deployment, mirroring the standard layout exactly. One documented mapping rule: where a prompt says `pm_skills/project/X`, this repo reads `self/project/X`; where it says root `AGENTS.md`, this repo reads `self/AGENTS.md`. Everything else — prompts, verbs, budgets, tiers — applies unchanged.
- **B — reshape `user_crud/` in place** (the ticket's original lean). Same idea, but keeps the meaningless `user_crud` name and conflicts with your "retire everything else, keep user_crud as archive" direction.
- **C — make `pm_skills/project/` the real memory** and move blank templates elsewhere. A breaking product restructure (major release, MANIFEST churn for all consumers) driven by this repo's convenience. Rejected.
